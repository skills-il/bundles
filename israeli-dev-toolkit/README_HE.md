# ערכת כלים למפתחים ישראלים

לבנות תוכנה לשוק הישראלי זה לא בדיוק כמו לבנות לשוק האמריקאי. יש פה דרישות שהפריימוורקים הבינלאומיים פשוט לא חושבים עליהן: פריסת RTL, טיפול נכון בטקסט עברי, אימות תעודת זהות, פירמוט מספרי טלפון מקומיים וכל מיני קוויקים של לוקליזציה. החבילה הזו נותנת לכם חמישה סקילי AI שפותרים בדיוק את משימות הפיתוח האלה.

## עובדות מפתח

- **תעודת זהות ישראלית**: **9 ספרות** עם ספרת ביקורת שמחושבת לפי אלגוריתם דמוי-Luhn. חשוב לאמת בצד השרת לפני שמשתמשים בזה כמפתח ראשי או בתהליכי KYC.
- **פורמט טלפון ישראלי**: סלולרי מתחיל ב-`05X` (050, 052, 053, 054, 055, 058). טלפון קווי משתמש בקידומות אזור `02, 03, 04, 08, 09`. פורמט בינלאומי הוא `+972` עם הסרת ה-0 הראשון.
- **ITU-T E.164**: הפורמט הקנוני המומלץ לאחסון מספרי טלפון (`+972501234567`).
- **מטבע**: ש"ח / ILS (שקל, `₪`, קוד ISO 4217: `ILS`), 2 ספרות אחרי הנקודה (אגורות).
- **קוד לוקאל**: `he-IL` לעברית (ישראל). `he` נתמך באופן רחב כפולבק.
- **אזור זמן**: `Asia/Jerusalem`, עם שעון קיץ (IDT = UTC+3 בקיץ, IST = UTC+2 בחורף).
- **מיקום סימן השקל**: `₪` בדרך כלל מופיע אחרי המספר בטקסט עברי (למשל `100 ₪`), לפני המספר בפורמט אנגלי.

## למי זה מתאים?

- **מפתחים** שבונים אפליקציות או אתרים למשתמשים בישראל
- **צוותים** שמוסיפים תמיכה בעברית או ב-RTL למוצרים שכבר קיימים
- **סטארטאפים** שמשיקים בשוק הישראלי
- **פרילנסרים** שעובדים עם לקוחות בארץ

## מה מקבלים

1. **בינאום לעברית**, הגדרת i18n מלאה, כולל תרגומים, פורמט תאריכים ומספרים והגדרות locale
2. **שיטות עבודה ל-RTL בעברית**, התכונות הלוגיות של CSS, טיפול בטקסט דו-כיווני ופיתוח בגישת RTL-first
3. **מאמת תעודת זהות ישראלית**, אימות מספרי תעודת זהות עם אלגוריתם ספרת הביקורת הנכון
4. **מפרמט טלפון ישראלי**, נרמול ופירמוט של כל סוגי מספרי הטלפון הישראליים לפורמט אחיד
5. **פריסט Tailwind לעברית**, הגדרה מוכנה של Tailwind CSS עם גופנים עבריים, כלי RTL ודפוסי עיצוב מקומיים
